# 社群成员信息收集表

> 用途：复制到飞书文档、飞书表单或飞书多维表格表单中，作为成员信息收集标准。所有字段名必须保持一致，后续 `import-member` 工作流会按这些字段映射到 `members/`。

## 表单说明

请用 3-5 分钟认真填写。这个表不是通讯录，而是为了帮社群成员更精准地互相连接：谁能帮你、你能帮谁、适合把你介绍给谁。

## 字段

### 1. 姓名/昵称

- 字段名：`name`
- 类型：单行文本
- 必填：是
- 说明：请填写社群内常用称呼，后续会作为成员档案文件名。

### 2. 所在城市

- 字段名：`city`
- 类型：单行文本
- 必填：是
- 示例：长沙

### 3. 所属社群/活动

- 字段名：`community`
- 类型：单行文本
- 必填：是
- 示例：长沙AI实战线下沙龙

### 4. 我是谁

- 字段名：`intro`
- 类型：多行文本
- 必填：是
- 说明：一句话讲清楚你的身份、行业、主要业务或当前阶段。

### 5. 我能提供什么

- 字段名：`offers`
- 类型：多行文本
- 必填：是
- 说明：请列出你能提供的资源、服务、经验、渠道、能力。建议一行一个。

### 6. 我正在找什么

- 字段名：`needs`
- 类型：多行文本
- 必填：是
- 说明：请列出你当前最想解决的问题、想找的客户、资源或合作。建议一行一个。

### 7. 我想链接谁

- 字段名：`looking_for`
- 类型：多行文本
- 必填：是
- 说明：请写具体人群，不要写“所有人”。例如企业主、活动主理人、财税顾问、AI开发者。

### 8. 我能帮社群做什么

- 字段名：`can_help_with`
- 类型：多行文本
- 必填：是
- 说明：写你愿意为社群贡献的事情，例如分享、答疑、组织活动、整理资料、提供场地、转介绍。

### 9. 适合参与/被推荐的场景

- 字段名：`scenarios`
- 类型：多行文本或多选
- 必填：是
- 说明：请写你最适合出现的具体场景。建议一行一个，例如企业内训、园区专场、AI自动化、短视频获客、CRM搭建、法律合规、场地合作。

### 10. 备注

- 字段名：`notes`
- 类型：多行文本
- 必填：否
- 说明：补充偏好、边界、合作注意事项。

## 可选字段

### 行业标签

- 字段名：`tags`
- 类型：多行文本
- 必填：否
- 说明：如果表单支持标签，可让成员自己填 3-5 个关键词；否则由 Agent 导入时生成。

### 角色类型

- 字段名：`role`
- 类型：单选
- 必填：否
- 选项：`frontstage-leader`, `backend-operator`, `business-resource`, `event-organizer`, `content-curator`, `connector`, `member`
- 说明：不建议普通成员自己填写，默认由 Agent 根据介绍判断。

### 成员阶段

- 字段名：`stage`
- 类型：单选
- 必填：否
- 选项：`seed`, `active`, `normal`, `silent`, `risk`
- 说明：不建议普通成员自己填写，默认由运营者或 Agent 判断。
